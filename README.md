# Zephyr Workbench ECLAIR Tools

Run ECLAIR static code analysis for Zephyr projects directly from the command line or GitHub Actions while reusing the configuration files generated by the [**ECLAIR Manager panel**](https://z-workbench.com/docs/documentation/static-code-analysis/eclair-manager) in the [Zephyr Workbench VS Code extension](https://z-workbench.com/).


## GitHub Action

Add the action to your workflow to reuse the same configuration in CI:

```yaml
jobs:
  analyze:
    runs-on: ubuntu-22.04
    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: '20'
    - uses: BUGSENG/zephyr-workbench-eclair-tools/cli@main
      id: run-analyze
      with:
        project-dir: '.'
        board: 'stm32f746g_disco'
        config: 'Application Only'
        venv-path: '../.venv'
    - uses: actions/upload-artifact@v4
      if: always()
      with:
        name: eclair-db
        path: ${{ steps.run-analyze.outputs.sca-dir }}
```

### Inputs

| Name | Required | Description |
| --- | --- | --- |
| `project-dir` | yes | Zephyr app folder containing `.vscode/zephyr-workbench.eclair.json`. |
| `board` | yes | Board or board/SoC triple passed to `west build`. |
| `config` | no | Name from the `sca_configs` array; falls back to `current_config_index`. |
| `build-dir` | no | Custom build directory for `west`. |
| `venv-path` | no | Python virtualenv to activate before running `west` (supports relative paths like `../.venv`). |
| `dry-run` | no | Set to `true` to print the command without running it. |
| `verbose` | | Set to `true` for more logs. |

### Outputs

- `sca-dir`: the absolute path to `build/sca/eclair`, suitable for `upload-artifact` or regression scripts.
